# 화면별 상세 설계서

문서명: 스마트 자동화 설비 통합 제어 소프트웨어 화면별 상세 설계서  
문서 버전: v0.1  
최종 수정일: 2026-04-14  
기준 문서: `PROJECT_PLAN.md`, `REQUIREMENTS_SPEC.md`

## 1. 문서 목적

본 문서는 1차 WPF 개발 범위에서 필요한 주요 화면의 구성, 표시 정보, 주요 동작, 향후 확장 포인트를 정의한다.

## 2. 공통 화면 구조

전체 애플리케이션은 다음 레이아웃을 기본으로 한다.

- 상단: 앱 제목, 현재 시간, 사용자 상태, 전체 장비 연결 요약
- 좌측: 메뉴 네비게이션
- 중앙: 선택된 화면의 본문 영역
- 하단 또는 우측: 상태바 또는 최근 이벤트 요약 영역

공통 메뉴 초안:

- 대시보드
- 검사 이력
- 알람 로그
- 레시피 관리
- 수동 제어
- 설정

## 3. 대시보드 화면

화면 목적:

- 시스템 전체 상태를 한눈에 보여주는 메인 화면

주요 구성:

- 설비 상태 카드
- 생산 집계 카드
- 최근 검사 결과 카드
- 장비 연결 상태 패널
- 최근 알람 패널
- 공정 상태 요약 패널

표시 항목:

- 현재 운전 상태
- 자동 / 수동 모드
- 총 생산 수량
- OK 수량
- NG 수량
- 마지막 검사 시각
- 마지막 판정 결과
- 장비별 연결 상태
- 최근 알람 메시지

주요 동작:

- 화면 진입 시 최신 상태를 조회한다.
- 상태 값 변경 시 카드와 패널이 갱신된다.
- 알람이 존재하면 강조 색상으로 표시한다.

향후 확장:

- 실시간 비전 결과 미리보기
- 로봇 상태 카드 추가
- I/O 맵 상태 요약 추가

## 4. 검사 이력 화면

화면 목적:

- 누적된 검사 결과를 목록으로 조회하고 검색하는 화면

주요 구성:

- 검색 조건 영역
- 검사 이력 목록 그리드
- 상세 정보 패널 또는 선택 행 정보 영역

검색 조건:

- 기간 시작일
- 기간 종료일
- 결과 조건(`전체`, `OK`, `NG`)
- 제품 ID 또는 키워드

그리드 컬럼 초안:

- 검사 시각
- 제품 ID
- 결과
- 판정값
- 비고

주요 동작:

- 검색 버튼 클릭 시 조건에 맞는 목록을 조회한다.
- 행 선택 시 상세 정보를 표시할 수 있다.
- 기본 정렬은 최신순이다.

향후 확장:

- 이미지 썸네일 표시
- 결과 CSV 내보내기
- 불량 유형 분류 필드 추가

## 5. 알람 로그 화면

화면 목적:

- 시스템 이벤트와 알람 기록을 확인하는 화면

주요 구성:

- 알람 필터 영역
- 알람 목록 그리드
- 상세 메시지 확인 영역

필터 항목:

- 기간
- 레벨(`정보`, `경고`, `오류`)
- 확인 여부

그리드 컬럼 초안:

- 발생 시각
- 레벨
- 구분
- 메시지
- 확인 여부

주요 동작:

- 알람 선택 시 상세 메시지를 표시한다.
- 확인 처리 버튼으로 상태를 변경할 수 있다.
- 최신 알람이 상단에 위치한다.

향후 확장:

- 장비별 알람 필터
- 에러 코드 분류
- 알람 음소거 또는 재통지 정책

## 6. 레시피 관리 화면

화면 목적:

- 제품별 검사 및 공정 설정값을 관리하는 화면

주요 구성:

- 좌측 레시피 목록
- 우측 상세 편집 폼
- 상단 또는 하단 저장/삭제 버튼 영역

목록 표시 항목:

- 레시피명
- 제품명
- 사용 여부

상세 입력 항목 초안:

- 레시피명
- 제품명
- 검사 기준값 1
- 검사 기준값 2
- 공정 옵션
- 사용 여부
- 비고

주요 동작:

- 레시피 선택 시 상세 정보가 로드된다.
- 신규 버튼으로 입력 폼을 초기화한다.
- 저장 시 신규 또는 수정으로 반영한다.
- 삭제 시 선택 레시피를 제거한다.

향후 확장:

- 장비별 세부 설정 탭
- 비전 파라미터 연동
- 로봇 동작 조건 매핑

## 7. 수동 제어 화면

화면 목적:

- 운영자가 직접 기본 제어 명령을 수행하는 화면

주요 구성:

- 공정 제어 버튼 영역
- 장비별 수동 조작 패널
- 현재 상태 표시 패널
- 이벤트 로그 미니 패널

제어 버튼:

- 시작
- 정지
- 리셋
- 자동 모드 전환
- 수동 모드 전환

장비별 패널 초안:

- Vision 상태 영역
- I/O 상태 영역
- Robot 상태 영역

현재 단계 동작:

- 버튼 입력 시 Mock 상태를 변경한다.
- 상태 변경 결과를 이벤트 로그에 남긴다.
- 위험한 동작은 확인 절차를 둘 수 있다.

향후 확장:

- Arduino / 페어리노 출력 ON/OFF
- Dobot 명령 전송
- 공정 단계 수동 진행

## 8. 설정 화면

화면 목적:

- 앱 기본 설정과 향후 통신 설정을 관리하는 화면

주요 구성:

- 데이터 저장 설정
- 통신 설정
- 시스템 옵션

입력 항목 초안:

- DB 경로
- 로그 저장 여부
- 장비 통신 사용 여부
- Serial 포트
- BaudRate
- TCP 주소 / 포트

주요 동작:

- 설정값 저장
- 기본값 복원
- 유효성 검증

향후 확장:

- 장비별 연결 테스트 버튼
- Jetson / 로봇 별도 설정 탭

## 9. 내비게이션 흐름

기본 사용자 흐름은 다음과 같다.

1. 사용자는 대시보드에서 전체 상태를 확인한다.
2. 필요 시 검사 이력 또는 알람 로그로 이동해 세부 내역을 본다.
3. 관리자 권한이 있으면 레시피 관리 화면에서 기준값을 수정한다.
4. 운영 중 문제가 발생하면 수동 제어 화면에서 대응한다.
5. 초기 환경 구성이나 통신 변경은 설정 화면에서 수행한다.

## 10. UI 설계 원칙

- 산업용 HMI 느낌의 명확한 정보 전달을 우선한다.
- 자주 보는 상태값은 카드 또는 배지 형태로 강조한다.
- 경고와 오류는 색상 체계를 분리한다.
- 클릭 동선은 짧고 일관되어야 한다.
- 향후 실시간 상태 갱신을 고려하여 과도한 복잡도를 피한다.

공통 고려사항:

- 명령 바인딩 구성
- Mock 서비스 주입 가능 구조

## 12. 향후 반영 예정 항목

- 실제 화면 와이어프레임
- 상태 색상 가이드
- 사용자 권한에 따른 버튼 활성/비활성 정책
- 반응형 크기 조정 기준
- 장비 연결 후 실시간 갱신 정책

## 13. 문서 관리 규칙

- 화면 추가 시 본 문서에 신규 섹션을 추가한다.
- 화면 구성 변경 시 표시 항목과 주요 동작을 함께 수정한다.
- 구현 단계에서 화면명이 바뀌면 ViewModel 명칭도 같이 정리한다.
