## 1. 프로젝트 개요

프로젝트명: `스마트 자동화 설비 통합 제어 소프트웨어`

본 프로젝트는 제조 자동화 환경을 가정한 PC 기반 운영 소프트웨어를 개발하는 프로젝트이다.  
1차 단계에서는 `C# / .NET / WPF` 기반의 데스크톱 애플리케이션을 중심으로 설비 상태 모니터링, 검사 결과 관리, 알람 로그, 레시피 관리, 수동 제어 기능을 구현한다.  
이후 단계적으로 `Jetson Orin Nano`, `OAK-D Lite`, `Arduino`, `페어리노`, `Dobot` 등 외부 장비를 연동하여 비전 검사, I/O 제어, 로봇 동작 제어까지 확장한다.

## 2. 프로젝트 목표

- WPF 기반 산업용 운영 소프트웨어 설계 및 구현
- 실시간 상태 모니터링과 검사 이력 관리 기능 구현
- 하드웨어 연동 전에도 동작 가능한 Mock 기반 구조 확보
- 향후 비전 장치, I/O 제어기, 로봇과 연동 가능한 확장형 아키텍처 설계
- 자동화 소프트웨어 직무 포트폴리오로 활용 가능한 수준의 결과물 확보

## 3. 추진 방향

현재 보유 장비는 다음과 같다.

- `Jetson Orin Nano`
- `OAK-D Lite`
- `Arduino`
- `페어리노`
- `Dobot`

단, 초기 개발은 장비 연동보다 `WPF 애플리케이션 완성도 확보`를 우선한다.  
즉, 실제 장비가 없어도 개발이 진행되도록 가상 데이터와 Mock 서비스로 먼저 기능을 구현하고, 이후 실제 장비 구현체를 추가하는 방식으로 진행한다.

## 4. 개발 단계

### 4.1 1차 단계

WPF 중심 운영 소프트웨어 개발

- 메인 대시보드
- 검사 이력 관리
- 알람/이벤트 로그
- 레시피 관리
- 수동 제어 화면
- 가상 설비 상태 표시
- 가상 검사 결과 생성
- 로컬 DB 저장

### 4.2 2차 단계

비전 및 I/O 장치 연동

- Jetson Orin Nano 연동
- OAK-D Lite 카메라 데이터 수신
- Arduino / 페어리노 기반 I/O 제어
- Serial 또는 TCP 통신 연결
- 실시간 검사 결과 반영

### 4.3 3차 단계

로봇 및 공정 자동화 연동

- Dobot 제어 연동
- 검사 결과에 따른 분류/이송 동작
- 공정 시퀀스 자동화
- 장비 간 상태 동기화

## 5. 시스템 컨셉

프로젝트의 최종 컨셉은 다음과 같다.

1. WPF 운영 소프트웨어가 전체 공정 상태를 통합 관리한다.
2. 비전 장치는 검사 데이터를 생성한다.
3. I/O 제어기는 센서 입력과 액추에이터 출력을 담당한다.
4. 로봇은 검사 결과에 따라 이송 또는 분류 작업을 수행한다.
5. 사용자는 PC 소프트웨어에서 전체 장비 상태를 모니터링하고 수동/자동 제어를 수행한다.

## 6. 주요 기능

### 6.1 설비 모니터링

- 설비 운전 상태 표시
- 자동 / 수동 / 정지 상태 전환
- 장비별 연결 상태 표시
- 생산 수량, 정상 수량, 불량 수량 표시

### 6.2 검사 결과 관리

- 검사 결과 `OK / NG` 표시
- 제품 ID, 검사 시각, 결과 이력 저장
- 조건별 검색 및 조회
- 일별 또는 모델별 집계 표시

### 6.3 알람 및 이벤트 로그

- 경고, 오류, 시스템 이벤트 기록
- 시간순 로그 조회
- 알람 발생 시 강조 표시
- 향후 장비별 에러 코드 분리 관리

### 6.4 레시피 관리

- 제품별 검사 조건 저장
- 공정별 설정값 저장
- 임계값 및 장비 옵션 설정
- 레시피 불러오기 / 수정 / 저장

### 6.5 수동 제어

- 시작 / 정지 / 리셋
- 장비별 수동 동작 버튼
- 추후 I/O 및 로봇 제어 명령과 연결 가능한 UI 구조 설계

### 6.6 통신 관리

- 현재는 Mock 서비스 기반
- 이후 Serial / TCP / REST 등으로 확장
- 장비별 통신 상태와 마지막 수신 시각 표시

## 9. 기술 스택

- Language: `C#`
- Framework: `.NET 8`
- UI: `WPF`
- Architecture: `MVVM`
- Database: `SQLite`
- Logging: `Serilog`

향후 확장 예정 기술:

- Jetson: `Python`, `OpenCV`, `DepthAI`
- Controller: `Serial Communication`, `TCP/IP`
- Robot: Dobot SDK 연동 방식 검토

## 11. 인터페이스 설계 원칙

초기에는 실제 장비 대신 Mock 구현체를 사용하고, 이후 실제 구현체로 교체 가능한 구조를 목표로 한다.

예상 인터페이스 예시:

- `IVisionService`
- `IIoControllerService`
- `IRobotService`
- `IAlarmService`
- `IRecipeService`

핵심 원칙:

- UI와 장비 제어 로직 분리
- 장비별 의존성 최소화
- 실제 장비 없는 환경에서도 테스트 가능
